# AGENTS.md

## 基本方針

- 変更対象ではない箇所では、改行・タブ・空白のみの差分を原則作らない。
- README.md 以外の Markdown は日本語で記述する。
- 関数引数・オプション引数は、明示要件がない限り安易に省略可にしない。
- タイムアウト値・待機上限・強制停止条件は、仕様として必要になるまで追加しない。
- 暫定回避や重複アーキテクチャを避け、既存の責務境界に統合する。

## このリポジトリの責務

- LLM クライアントではなく、ローカル推論エンジンの最小実装を扱う。
- llama.cpp に近い責務分割を保つ。
  - `Model`: 重みとメタデータ
  - `Context`: 推論状態
  - `Backend`: テンソル演算の実行先
  - `Sampler`: 次トークン選択
  - `Engine`: 評価ループ
- AtomVM 想定のため、コアは純 Elixir で動く構造を優先する。
- Nx / EXLA は差し替え可能なバックエンドとして扱い、必須依存にしない。

## 作業手順

- Beads issue は内部の作業管理として使う。
- コード修正後は `mix test` を実行する。
- 差分説明は、全体要約を先に述べてからファイルごとの要点を述べる。
